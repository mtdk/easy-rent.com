## 1. 使用说明

- 执行环境：MySQL / MariaDB
- 目标库：easy_rent
- 执行方式：在每次导入完成后按顺序执行本清单 SQL
- 通过标准：各检查项结果满足“期望值”

示例连接命令：

```bash
mysql -h127.0.0.1 -P3306 -uxmtdk -p12345678 -D easy_rent
```

## 2. 规模与覆盖检查

### 2.1 核心表规模

```sql
SELECT COUNT(*) AS users_cnt FROM users;
SELECT COUNT(*) AS properties_cnt FROM properties;
SELECT COUNT(*) AS contracts_cnt FROM contracts;
SELECT COUNT(*) AS payments_cnt FROM rent_payments;
SELECT COUNT(*) AS financial_records_cnt FROM financial_records;
```

期望：
- `payments_cnt` 与 `financial_records_cnt` 同步增长，历史导入批次应保持一致口径。

### 2.2 覆盖范围

```sql
SELECT COUNT(DISTINCT c.tenant_name) AS tenant_cnt
FROM rent_payments rp
JOIN contracts c ON c.id = rp.contract_id;

SELECT COUNT(DISTINCT p.property_code) AS property_cnt
FROM rent_payments rp
JOIN contracts c ON c.id = rp.contract_id
JOIN properties p ON p.id = c.property_id;

SELECT MIN(payment_period) AS min_period, MAX(payment_period) AS max_period
FROM rent_payments;
```

期望：
- `tenant_cnt`、`property_cnt` 不为 0 且与导入批次规模匹配。
- `min_period/max_period` 落在预期历史区间。

## 3. 结构与约束检查

### 3.1 账单状态一致性

```sql
SELECT COUNT(*) AS non_paid_status_cnt
FROM rent_payments
WHERE payment_status <> 'paid';
```

期望：
- 历史“已收款导入”场景下，结果为 0。

### 3.2 金额合法性

```sql
SELECT COUNT(*) AS non_positive_amount_cnt
FROM rent_payments
WHERE amount_due <= 0 OR amount_paid <= 0;
```

期望：
- 结果为 0。

### 3.3 账期格式合法性

```sql
SELECT COUNT(*) AS period_format_invalid_cnt
FROM rent_payments
WHERE payment_period NOT REGEXP '^[0-9]{4}-[0-9]{2}$';
```

期望：
- 结果为 0。

### 3.4 账单与财务流水引用完整性

```sql
SELECT COUNT(*) AS missing_financial_ref_cnt
FROM rent_payments rp
LEFT JOIN financial_records fr
  ON fr.reference_type = 'rent_payment'
 AND fr.reference_id = rp.id
WHERE fr.id IS NULL;
```

期望：
- 历史导入方案为“账单+流水成对写入”时，结果为 0。

## 4. 重复导入风险检查

### 4.1 同合同同周期重复账单

```sql
SELECT contract_id, payment_period, COUNT(*) AS dup_cnt
FROM rent_payments
GROUP BY contract_id, payment_period
HAVING COUNT(*) > 1
ORDER BY dup_cnt DESC, contract_id ASC, payment_period ASC;
```

期望：
- 无结果集（0 行）。

### 4.2 支付编号唯一性检查

```sql
SELECT payment_number, COUNT(*) AS dup_cnt
FROM rent_payments
GROUP BY payment_number
HAVING COUNT(*) > 1
ORDER BY dup_cnt DESC, payment_number ASC;
```

期望：
- 无结果集（0 行）。

## 6. 分布检查（防止导入倾斜）

### 6.1 按房号分布 Top N

```sql
SELECT p.property_code, COUNT(*) AS payment_rows
FROM rent_payments rp
JOIN contracts c ON c.id = rp.contract_id
JOIN properties p ON p.id = c.property_id
GROUP BY p.property_code
ORDER BY payment_rows DESC, p.property_code ASC
LIMIT 20;
```

### 6.2 按租客分布 Top N

```sql
SELECT c.tenant_name, COUNT(*) AS payment_rows
FROM rent_payments rp
JOIN contracts c ON c.id = rp.contract_id
GROUP BY c.tenant_name
ORDER BY payment_rows DESC, c.tenant_name ASC
LIMIT 20;
```

期望：
- 分布与历史业务认知一致；若出现异常尖峰，回看源数据与解析规则。
