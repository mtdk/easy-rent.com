# A.xlsx 分析报告（2026-04-06）

> 文档状态：历史（归档参考）
> 口径说明：以 docs/主任务看板.md 与 docs/任务清单.md 为准。


## 1. 文件总体结构

- 文件：docs/A.xlsx
- 工作表数量：23（按房号拆分的租金台账 + 汇总辅助页）
- 典型工作表：
  - 房号页：9-102#、9-1-101#、29-201# ...
  - 辅助页：支出统计、退租记录、综合类别

从工作簿元数据看，这份表是“按房间维护”的历史台账，结构与你当前系统“合同/账单/收款”模型高度匹配。

## 2. 房号页核心字段（与你业务核心一致）

在房号页（如 sheet3）中，首行与次行可解码为以下字段：

- 日期
- 房号
- 用户名（租客）
- 水表（当前度数/上次度数）
- 电表（当前度数/上次度数）
- 用水量
- 用电量
- 水费
- 电费
- 租金
- 合计
- 押金
- 备注

其中关键公式关系已在表内体现：

- 用水量 = 水表当前度数 - 水表上次度数
- 用电量 = 电表当前度数 - 电表上次度数
- 水费 = 用水量 * 水费单价
- 电费 = 用电量 * 电费单价
- 合计 = 租金 + 水费 + 电费（可再叠加历史余额/优惠）

这与当前系统已落地的“水电+租金账单”逻辑一致。

## 3. 其他工作表价值

### 3.1 支出统计

表头可识别为：

- 日期
- 项目
- 金额
- 类别

可用于你当前系统的 financial_records（expense）导入或作为独立“维护支出台账”。

### 3.2 退租记录

表头与支出页一致风格，含：

- 日期
- 项目（如退某房押金/房租）
- 金额
- 类别（如退押金/房租）

可映射为“租客退租结算记录”。

## 4. 与当前系统字段映射建议

### 4.1 房号页 -> rent_payments

- 日期 -> payment_period + due_date（按月规则推导）
- 房号 -> 通过 property_name/contract 关联 contract_id
- 用户名 -> tenant_name（用于匹配合同）
- 租金 -> amount_due 的 rent_amount 分量
- 水表上次/当前 + 电表上次/当前 -> notes.formula.*
- 用量/水费/电费 -> notes.formula.*
- 合计 -> amount_due
- 备注 -> notes.payment_note（或附加备注）

### 4.2 支出统计/退租记录 -> financial_records

- 日期 -> transaction_date
- 项目 -> description
- 金额 -> amount
- 类别 -> category
- 记录类型 -> record_type=expense（退押金可用 expense/adjustment）

## 5. 导入时的数据清洗规则（必须）

1. 日期标准化：
- Excel序列日期（如44933）需转 YYYY-MM-DD。

2. 金额与度数清洗：
- 去除中文说明中混杂数字（如“实收260元”）与纯文本备注分离。
- 数字字段保留两位小数，空值按 NULL。

3. 身份匹配策略：
- 优先按“房号+租客名+周期”匹配合同。
- 未匹配到合同的数据先入“待确认队列”，不要直接入正式账单。

4. 备注保留：
- 对“多收/少收、优惠、搬离结算”等说明保留原文，后续可审计。

## 6. 你当前系统可直接复用的点

你现有系统已经具备：

- 水电读数计算账单
- 自动带出上期读数
- 收款登记
- CSV导出

所以 A.xlsx 的导入不需要重做业务，只需要补“导入器 + 清洗规则”。
