## 1. 项目概述

### 1.1 项目背景
随着房地产租赁市场的快速发展，房东和物业管理公司需要一个高效、智能的收租管理系统来管理多个房产的租赁合同、租金收取、租客信息、费用统计等业务。传统的手工记录方式效率低下，容易出错，无法满足现代化管理的需求。

**特别需求**：
1. 系统需要在无互联网连接的内部局域网环境中运行，所有外部资源必须本地化部署
2. 第一期仅支持管理员和房东角色，租客功能作为未来扩展项保留接口

### 1.2 项目目标
- 实现租赁合同的数字化管理
- 自动化租金收取提醒和催缴
- 提供多维度数据统计和分析
- 支持局域网环境下的稳定运行
- 确保数据安全和隐私保护
- 所有前端资源本地化，不依赖外部CDN
- 设计可扩展架构，支持未来租客功能扩展

### 1.3 技术特点
- **纯局域网部署**：无需互联网连接，所有资源本地化
- **现代化技术栈**：PHP 8.2 + MariaDB + Bootstrap 5.3
- **响应式设计**：适配桌面和移动设备
- **离线功能**：支持无网络环境下的完整操作
- **易于维护**：结构清晰，部署简单
- **可扩展设计**：预留租客功能接口，支持未来扩展

## 2. 系统需求分析

### 2.1 功能需求（第一期：管理员+房东）

#### 2.1.1 用户管理
- **管理员账户管理**：系统最高权限，管理所有功能
- **房东账户管理**：房产所有者，管理自有房产
- **角色权限控制**：管理员、房东两级权限体系
- **登录/注册/密码找回**：无第三方登录，仅账号密码
- **租客功能**：作为未来扩展项，当前版本预留接口

#### 2.1.2 房产管理
- 房产信息录入（地址、面积、户型、设施等）
- 房产图片上传（本地存储）
- 房产状态管理（空闲、已租、维修中）
- 文本地址管理（无地图服务）
- 房产搜索与筛选

#### 2.1.3 合同管理
- 租赁合同创建、编辑、终止
- 合同模板管理
- 合同扫描件上传（替代电子签名）
- 合同到期提醒
- 续约管理
- 租客信息管理（作为合同关联方，非独立用户）

#### 2.1.4 租金管理
- 租金设置（月租、季租、年租）
- 租金收取记录
- 自动生成租金账单
- 逾期费用计算
- 线下支付记录（银行转账、现金等）
- 支付凭证上传

#### 2.1.5 财务管理
- 收入统计报表
- 支出记录（维修、水电等）
- 利润分析
- 财务报表导出（Excel/PDF）

#### 2.1.6 通知提醒
- 租金到期提醒（站内通知）
- 合同到期提醒
- 维修提醒
- 系统通知
- 局域网邮件通知（可选）

#### 2.1.7 报表分析
- 出租率统计
- 租金收入趋势分析
- 租客信息统计（基于合同数据）
- 空置期分析
- 投资收益分析

### 2.2 非功能需求（局域网环境）
- **网络环境**：纯局域网，无互联网连接
- **资源本地化**：Bootstrap、Icons等全部下载到本地
- **响应时间**：页面加载<3秒，操作响应<1秒
- **并发用户**：支持50+用户同时在线（局域网规模）
- **数据安全**：符合内部安全规范
- **可用性**：99.9%系统可用性
- **浏览器兼容**：现代浏览器（Chrome 60+, Firefox 60+, Safari 12+）
- **扩展性**：支持未来租客功能无缝扩展

## 3. 系统架构设计（更新）

- **包管理**: Composer (PHP)
- **本地开发**: XAMPP/MAMP 或 Docker

## 5. 数据库设计（MariaDB）

### 5.1 核心数据表
1. **users** - 用户表（管理员、房东）
2. **properties** - 房产表
3. **contracts** - 合同表（包含租客信息）
4. **invoices** - 发票/账单表
5. **payments** - 支付记录表
6. **maintenance_requests** - 维修请求表
7. **notifications** - 通知表
8. **financial_records** - 财务记录表

### 5.2 数据库ER图概要
```
users (admin/landlord) ──── properties ───── contracts ───── tenant_info
          │                     │                │
          │                     │                │
          │                maintenance_requests  │         payments
          │                     │                │              │
          │                     │                │         invoices
          │                     │                │              │
          └──── notifications ──┴────────────────┴──── financial_records
```

### 5.3 关键表设计要点
```sql
-- 用户表：当前仅支持admin和landlord，保留tenant枚举值用于未来扩展
CREATE TABLE users (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    -- ... 其他字段
    role ENUM('admin', 'landlord', 'tenant') DEFAULT 'landlord',
    -- tenant角色当前未使用，保留用于未来扩展
    ...
);

-- 合同表：租客信息直接存储，预留用户关联字段
CREATE TABLE contracts (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    property_id INT UNSIGNED NOT NULL,
    landlord_id INT UNSIGNED NOT NULL,
    -- 租客信息直接存储（当前版本）
    tenant_name VARCHAR(100) NOT NULL,
    tenant_phone VARCHAR(20),
    tenant_id_card VARCHAR(50),
    tenant_email VARCHAR(100),
    -- 预留字段：未来关联用户表
    tenant_user_id INT UNSIGNED NULL,
    ...
);
```
