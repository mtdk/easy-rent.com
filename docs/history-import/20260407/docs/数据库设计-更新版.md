# 收租管理系统数据库设计（更新版）

> 文档状态：历史（归档参考）
> 口径说明：以 docs/主任务看板.md 与 docs/任务清单.md 为准。


## 概述

本数据库设计针对收租管理系统第一期需求，支持管理员和房东两种角色，租客功能作为未来扩展项。数据库使用MariaDB 10.6+，采用UTF8MB4字符集，支持完整的租赁管理业务流程。

## 设计原则

1. **数据完整性**：使用外键约束确保数据一致性
2. **性能优化**：合理设计索引，优化查询性能
3. **扩展性**：预留租客功能扩展字段
4. **安全性**：敏感数据加密存储
5. **可维护性**：清晰的表结构和字段命名

## 数据库表清单

### 核心业务表
1. `users` - 用户表（管理员、房东）
2. `properties` - 房产表
3. `contracts` - 合同表（集成租客信息）
4. `invoices` - 账单表
5. `payments` - 支付记录表

### 支持表
6. `property_images` - 房产图片表

8. `maintenance_requests` - 维修请求表
9. `notifications` - 通知表

### 系统表

12. `login_logs` - 登录日志表
13. `system_settings` - 系统设置表
14. `audit_logs` - 审计日志表

## 详细表结构

### 1. 用户表 (users)

存储系统用户信息，当前仅支持管理员和房东角色。

```sql
CREATE TABLE users (
    -- 主键和标识
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    uuid CHAR(36) NOT NULL UNIQUE DEFAULT (UUID()),
    
    -- 登录信息
    username VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
    email VARCHAR(100) UNIQUE COMMENT '邮箱',
    phone VARCHAR(20) UNIQUE COMMENT '手机号',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希',
    
    -- 个人信息
    full_name VARCHAR(100) NOT NULL COMMENT '姓名',
    avatar VARCHAR(255) COMMENT '头像路径',
    gender ENUM('male', 'female', 'other') COMMENT '性别',
    birth_date DATE COMMENT '出生日期',
    
    -- 角色和状态
    role ENUM('admin', 'landlord', 'tenant') DEFAULT 'landlord' COMMENT '用户角色',
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否激活',
    is_verified BOOLEAN DEFAULT FALSE COMMENT '是否验证',
    
    -- 安全相关
    login_attempts INT DEFAULT 0 COMMENT '登录尝试次数',
    locked_until TIMESTAMP NULL COMMENT '锁定直到',
    last_login_at TIMESTAMP NULL COMMENT '最后登录时间',
    password_changed_at TIMESTAMP NULL COMMENT '密码修改时间',
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted_at TIMESTAMP NULL COMMENT '软删除时间',
    
    -- 索引
    INDEX idx_role (role),
    INDEX idx_email (email),
    INDEX idx_phone (phone),
    INDEX idx_is_active (is_active),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

**字段说明**：
- `role`：当前仅使用`admin`和`landlord`，`tenant`保留用于未来扩展
- `is_active`：控制用户是否可用
- `deleted_at`：软删除标记，支持数据恢复

### 2. 房产表 (properties)

存储房产信息，关联房东用户。

```sql
CREATE TABLE properties (
    -- 主键和标识
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    uuid CHAR(36) NOT NULL UNIQUE DEFAULT (UUID()),
    
    -- 所属关系
    owner_id INT UNSIGNED NOT NULL COMMENT '房东ID',
    
    -- 基本信息
    title VARCHAR(200) NOT NULL COMMENT '房产标题',
    description TEXT COMMENT '详细描述',
    
    -- 地址信息
    address TEXT NOT NULL COMMENT '详细地址',
    city VARCHAR(100) COMMENT '城市',
    district VARCHAR(100) COMMENT '区县',
    street VARCHAR(200) COMMENT '街道',
    building_no VARCHAR(50) COMMENT '楼号',
    unit_no VARCHAR(50) COMMENT '单元号',
    room_no VARCHAR(50) COMMENT '房号',
    postal_code VARCHAR(20) COMMENT '邮政编码',
    
    -- 房产属性
    property_type ENUM('apartment', 'house', 'office', 'shop', 'warehouse', 'other') 
        DEFAULT 'apartment' COMMENT '房产类型',
    area DECIMAL(10, 2) COMMENT '面积(平方米)',
    bedrooms INT COMMENT '卧室数量',
    bathrooms INT COMMENT '卫生间数量',
    living_rooms INT COMMENT '客厅数量',
    kitchens INT COMMENT '厨房数量',
    
    -- 楼层信息
    floor INT COMMENT '所在楼层',
    total_floors INT COMMENT '总楼层',
    orientation VARCHAR(50) COMMENT '朝向',
    decoration ENUM('rough', 'simple', 'hard', 'luxury') DEFAULT 'simple' COMMENT '装修情况',
    
    -- 设施信息（JSON格式）
    facilities JSON COMMENT '设施列表',
    
    -- 租赁信息
    status ENUM('vacant', 'rented', 'maintenance', 'sold', 'other') DEFAULT 'vacant' COMMENT '状态',
    monthly_rent DECIMAL(12, 2) COMMENT '月租金',
    deposit DECIMAL(12, 2) COMMENT '押金',
    minimum_lease_months INT DEFAULT 12 COMMENT '最短租期(月)',
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    -- 外键约束
    FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE CASCADE,
    
    -- 索引
    INDEX idx_owner (owner_id),
    INDEX idx_status (status),
    INDEX idx_city (city),
    INDEX idx_property_type (property_type),
    INDEX idx_monthly_rent (monthly_rent),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='房产表';
```

### 3. 合同表 (contracts)

存储租赁合同信息，集成租客信息（非独立用户）。

```sql
CREATE TABLE contracts (
    -- 主键和标识
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    uuid CHAR(36) NOT NULL UNIQUE DEFAULT (UUID()),
    
    -- 关联信息
    property_id INT UNSIGNED NOT NULL COMMENT '房产ID',
    landlord_id INT UNSIGNED NOT NULL COMMENT '房东ID',
    
    -- 租客信息（直接存储，当前版本）
    tenant_name VARCHAR(100) NOT NULL COMMENT '租客姓名',
    tenant_phone VARCHAR(20) NOT NULL COMMENT '租客电话',
    tenant_id_card VARCHAR(50) COMMENT '租客身份证号',
    tenant_email VARCHAR(100) COMMENT '租客邮箱',
    tenant_address TEXT COMMENT '租客地址',
    tenant_emergency_contact VARCHAR(100) COMMENT '紧急联系人',
    tenant_emergency_phone VARCHAR(20) COMMENT '紧急联系电话',
    
    -- 预留字段：未来关联用户表
    tenant_user_id INT UNSIGNED NULL COMMENT '租客用户ID（未来扩展）',
    
    -- 合同基本信息
    contract_number VARCHAR(100) UNIQUE NOT NULL COMMENT '合同编号',
    title VARCHAR(200) NOT NULL COMMENT '合同标题',
    content TEXT NOT NULL COMMENT '合同内容',
    template_id INT UNSIGNED COMMENT '模板ID',
    
    -- 租赁期限
    start_date DATE NOT NULL COMMENT '开始日期',
    end_date DATE NOT NULL COMMENT '结束日期',
    actual_end_date DATE COMMENT '实际结束日期',
    
    -- 费用信息
    monthly_rent DECIMAL(12, 2) NOT NULL COMMENT '月租金',
    deposit DECIMAL(12, 2) NOT NULL COMMENT '押金',
    agency_fee DECIMAL(12, 2) DEFAULT 0 COMMENT '中介费',
    other_fees DECIMAL(12, 2) DEFAULT 0 COMMENT '其他费用',
    
    -- 付款信息
    payment_day INT DEFAULT 1 COMMENT '每月付款日',
    payment_method ENUM('bank_transfer', 'cash', 'check', 'other') DEFAULT 'bank_transfer' COMMENT '付款方式',
    bank_account_name VARCHAR(100) COMMENT '收款账户名',
    bank_account_number VARCHAR(50) COMMENT '收款账号',
    bank_name VARCHAR(100) COMMENT '开户行',
    
    -- 合同状态
    status ENUM('draft', 'pending', 'active', 'expired', 'terminated', 'cancelled') 
        DEFAULT 'draft' COMMENT '合同状态',
    
    -- 文件相关
    scan_file_path VARCHAR(500) COMMENT '扫描件路径',
    scan_file_name VARCHAR(255) COMMENT '扫描件文件名',
    
    -- 签署信息
    signed_at TIMESTAMP NULL COMMENT '签署时间',
    signed_location VARCHAR(200) COMMENT '签署地点',
    
    -- 终止信息
    terminated_at TIMESTAMP NULL COMMENT '终止时间',
    termination_reason TEXT COMMENT '终止原因',
    termination_type ENUM('landlord', 'tenant', 'mutual', 'expired') COMMENT '终止类型',
    
    -- 备注
    notes TEXT COMMENT '备注',
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    -- 外键约束
    FOREIGN KEY (property_id) REFERENCES properties(id),
    FOREIGN KEY (landlord_id) REFERENCES users(id),
    FOREIGN KEY (template_id) REFERENCES contract_templates(id),
    FOREIGN KEY (tenant_user_id) REFERENCES users(id) ON DELETE SET NULL,
    
    -- 索引
    INDEX idx_property (property_id),
    INDEX idx_landlord (landlord_id),
    INDEX idx_tenant_phone (tenant_phone),
    INDEX idx_contract_number (contract_number),
    INDEX idx_status (status),
    INDEX idx_start_date (start_date),
    INDEX idx_end_date (end_date),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='合同表';
```

**扩展说明**：
- `tenant_user_id`：当前为NULL，未来租客功能扩展时关联users表
- `contract_number`：唯一合同编号，格式如`CONTRACT-20240405-ABC123`
- 租客信息直接存储，便于当前版本使用

### 4. 账单表 (invoices)

存储租金账单信息。

```sql
CREATE TABLE invoices (
    -- 主键和标识
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    uuid CHAR(36) NOT NULL UNIQUE DEFAULT (UUID()),
    
    -- 关联信息
    contract_id INT UNSIGNED NOT NULL COMMENT '合同ID',
    
    -- 账单信息
    invoice_number VARCHAR(100) UNIQUE NOT NULL COMMENT '账单编号',
    period_start DATE NOT NULL COMMENT '账单周期开始',
    period_end DATE NOT NULL COMMENT '账单周期结束',
    
    -- 费用明细
    base_amount DECIMAL(12, 2) NOT NULL COMMENT '基础租金',
    utilities_amount DECIMAL(12, 2) DEFAULT 0 COMMENT '水电费',
    other_charges DECIMAL(12, 2) DEFAULT 0 COMMENT '其他费用',
    late_fee DECIMAL(12, 2) DEFAULT 0 COMMENT '滞纳金',
    discount_amount DECIMAL(12, 2) DEFAULT 0 COMMENT '折扣金额',
    total_amount DECIMAL(12, 2) NOT NULL COMMENT '总金额',
    
    -- 付款信息
    due_date DATE NOT NULL COMMENT '到期日',
    paid_at TIMESTAMP NULL COMMENT '支付时间',
    payment_method ENUM('bank_transfer', 'cash', 'check', 'other') COMMENT '支付方式',
    payment_proof_path VARCHAR(500) COMMENT '支付凭证路径',
    
    -- 状态
    status ENUM('pending', 'paid', 'overdue', 'cancelled', 'refunded') DEFAULT 'pending' COMMENT '状态',
    
    -- 备注
    notes TEXT COMMENT '备注',
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    -- 外键约束
    FOREIGN KEY (contract_id) REFERENCES contracts(id),
    
    -- 索引
    INDEX idx_contract (contract_id),
    INDEX idx_invoice_number (invoice_number),
    INDEX idx_status (status),
    INDEX idx_due_date (due_date),
    INDEX idx_period_start (period_start),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='账单表';
```

### 5. 支付记录表 (payments)

存储详细的支付记录。

```sql
CREATE TABLE payments (
    -- 主键和标识
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    uuid CHAR(36) NOT NULL UNIQUE DEFAULT (UUID()),
    
    -- 关联信息
    invoice_id INT UNSIGNED NOT NULL COMMENT '账单ID',
    contract_id INT UNSIGNED NOT NULL COMMENT '合同ID',
    
    -- 支付信息
    amount DECIMAL(12, 2) NOT NULL COMMENT '支付金额',
    payment_method ENUM('bank_transfer', 'cash', 'check', 'other') NOT NULL COMMENT '支付方式',
    transaction_reference VARCHAR(200) COMMENT '交易参考号',
    payer_name VARCHAR(100) COMMENT '付款人姓名',
    payer_account VARCHAR(100) COMMENT '付款账号',
    
    -- 状态
    status ENUM('pending', 'completed', 'failed', 'refunded', 'cancelled') DEFAULT 'pending' COMMENT '状态',
    
    -- 时间信息
    paid_at TIMESTAMP NULL COMMENT '支付时间',
    confirmed_at TIMESTAMP NULL COMMENT '确认时间',
    refunded_at TIMESTAMP NULL COMMENT '退款时间',
    
    -- 备注和元数据
    notes TEXT COMMENT '备注',
    metadata JSON COMMENT '元数据',
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    -- 外键约束
    FOREIGN KEY (invoice_id) REFERENCES invoices(id),
    FOREIGN KEY (contract_id) REFERENCES contracts(id),
    
    -- 索引
    INDEX idx_invoice (invoice_id),
    INDEX idx_contract (contract_id),
    INDEX idx_status (status),
    INDEX idx_paid_at (paid_at),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='支付记录表';
```

## 支持表结构

### 6. 房产图片表 (property_images)

```sql
CREATE TABLE property_images (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    property_id INT UNSIGNED NOT NULL,
    image_path VARCHAR(500) NOT NULL,
    image_name VARCHAR(255) NOT NULL,
    file_size INT,
    mime_type VARCHAR(100),
    is_primary BOOLEAN DEFAULT FALSE,
    sort_order INT DEFAULT 0,
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (property_id) REFERENCES properties(id) ON DELETE CASCADE,
    INDEX idx_property (property_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 7. 合同模板表 (contract_templates)

```sql
CREATE TABLE contract_templates (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    content TEXT NOT NULL,
    variables JSON,
    is_default BOOLEAN DEFAULT FALSE,
    created_by INT UNSIGNED,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (created_by) REFERENCES users(id),
    INDEX idx_is_default (is_default)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 8. 维修请求表 (maintenance_requests)

```sql
CREATE TABLE maintenance_requests (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    property_id INT UNSIGNED NOT NULL,
    contract_id INT UNSIGNED,
    reported_by INT UNSIGNED COMMENT '报告人',
    title VARCHAR(200) NOT NULL,
    description TEXT,
    priority ENUM('low', 'medium', 'high', 'urgent') DEFAULT 'medium',
    status ENUM('pending', 'in_progress', 'completed', 'cancelled') DEFAULT 'pending',
    assigned_to INT UNSIGNED COMMENT '指派给',
    completed_at TIMESTAMP NULL,
    cost DECIMAL(10, 2) DEFAULT 0,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (property_id) REFERENCES properties(id),
    FOREIGN KEY (contract_id) REFERENCES contracts(id),
    FOREIGN KEY (reported_by) REFERENCES users(id),
    FOREIGN KEY (assigned_to) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
