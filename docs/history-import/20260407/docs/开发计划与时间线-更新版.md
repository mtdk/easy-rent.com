## 3. 开发重点调整

### 3.1 简化开发内容
1. **用户系统简化**：
   - 移除租客注册、登录功能
   - 简化权限矩阵（仅管理员、房东）
   - 移除租客个人中心界面

2. **合同管理调整**：
   - 租客信息作为合同字段存储
   - 移除租客用户关联逻辑
   - 简化租客信息管理界面

3. **通知系统简化**：
   - 仅支持管理员↔房东通知
   - 移除租客相关通知模板
   - 简化通知权限控制

### 3.2 扩展接口开发
1. **数据库扩展点**：
   - 用户表保留`tenant`角色枚举
   - 合同表预留`tenant_user_id`字段
   - 添加扩展标记注释

2. **代码扩展点**：
   - 预留租客相关路由前缀
   - 添加`// TODO: 租客功能扩展`注释
   - 设计可扩展的权限检查接口

3. **界面扩展点**：
   - 登录页面预留租客登录位置
   - 导航菜单预留租客功能入口
   - 样式设计考虑未来扩展

## 6. 质量保证计划

### 6.1 代码质量
- **代码规范**: 遵循PSR标准，统一编码风格
- **代码审查**: 关键代码必须经过审查
- **扩展注释**: 所有扩展点必须有详细注释
- **文档完整性**: API文档、部署文档、扩展文档

### 6.2 测试策略
| 测试类型 | 测试工具 | 测试范围 | 执行频率 |
|----------|----------|----------|----------|
| 单元测试 | PHPUnit | 核心类和方法 | 每次提交 |
| 功能测试 | 手动测试 | 用户操作流程（两级权限） | 每日构建 |
| 权限测试 | 自动化测试 | 权限边界测试 | 每周 |
| 性能测试 | Apache Bench | 响应时间和并发 | 每版本 |
| 扩展接口测试 | 手动测试 | 扩展点可用性 | 每版本 |

### 6.3 部署质量
- **预发布测试**: 在生产环境类似环境中测试
- **回滚计划**: 出现问题快速回滚到上一版本
- **监控设置**: 错误日志监控，性能监控
- **用户培训**: 提供管理员和房东操作培训
