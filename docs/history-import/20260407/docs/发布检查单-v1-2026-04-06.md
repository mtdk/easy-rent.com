## 4. 回滚预案

- [ ] 确认本次发布前备份文件
- [ ] 执行总控预演：`./scripts/run-restore-and-verify.sh preview storage/backups/<backup_file>.sql <base_url> <db_user> <db_password> <db_name> <db_host> <db_port>`
- [ ] 必要时执行总控提交：`./scripts/run-restore-and-verify.sh commit storage/backups/<backup_file>.sql <base_url> <db_user> <db_password> <db_name> <db_host> <db_port>`
- [ ] 保存总控执行日志：`storage/logs/restore-verify_<mode>_<timestamp>.log`
- [ ] 保存总控结构化报告：`storage/logs/restore-verify_<mode>_<timestamp>.json`
- [ ] 先执行恢复预览：`./scripts/restore-easy-rent.sh storage/backups/<backup_file>.sql <db_user> <db_password> <db_name> <db_host> <db_port> --dry-run`
- [ ] 执行一键恢复：`./scripts/restore-easy-rent.sh storage/backups/<backup_file>.sql <db_user> <db_password> <db_name> <db_host> <db_port> --force`
- [ ] 执行数据库恢复命令：

```bash
mysql -h<db_host> -P<db_port> -u<db_user> -p<db_password> <db_name> < storage/backups/<backup_file>.sql
```

- [ ] 恢复后执行关键冒烟验证
- [ ] 执行恢复后健康检查：`./scripts/post-restore-health-check.sh <base_url> <db_user> <db_password> <db_name> <db_host> <db_port>`
- [ ] 检查恢复历史记录：`./scripts/show-restore-history.sh 20 --failed-only`
- [ ] 输出恢复历史汇总：`./scripts/show-restore-history.sh 50 --summary`
- [ ] 导出恢复历史 CSV（可选）：`./scripts/show-restore-history.sh 50 --status=all --mode=all --csv=storage/logs/restore-history.csv`
- [ ] 导出恢复历史 JSON（可选）：`./scripts/show-restore-history.sh 50 --status=failed --mode=commit --json`
- [ ] 落盘恢复历史 JSON 文件（可选）：`./scripts/show-restore-history.sh 50 --status=all --mode=all --json-file=storage/logs/restore-history.json`
- [ ] 预演并执行历史清理（可选）：`./scripts/cleanup-restore-history.sh preview 30 50` / `./scripts/cleanup-restore-history.sh commit 30 50`
- [ ] 记录故障原因与回滚时间点
