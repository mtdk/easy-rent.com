## 1. 技术栈选择（基于新需求）

### 1.1 后端技术栈
- **编程语言**: PHP 8.2+
- **Web框架**: 原生PHP + 自定义MVC架构 或 Slim Framework/Laravel Lumen
- **数据库**: MariaDB 10.6+ (MySQL兼容)
- **缓存**: Redis 6+ (可选，用于性能优化)
- **会话管理**: PHP原生Session或Redis Session
- **文件存储**: 本地文件系统（局域网环境）

### 1.2 前端技术栈
- **HTML/CSS**: HTML5, CSS3
- **JavaScript**: 原生JavaScript + 少量jQuery（可选）
- **CSS框架**: Bootstrap 5.3.0（本地化部署）
- **图标库**: Bootstrap Icons 1.11.0（本地化部署）
- **前端构建**: 无需构建工具，直接使用CDN替代为本地文件

### 1.3 开发环境
- **Web服务器**: Apache 2.4+ 或 Nginx 1.20+
- **PHP扩展**: PDO, MySQLi, JSON, MBString, OpenSSL, GD
- **开发工具**: VS Code/PHPStorm, Git
- **本地开发**: XAMPP/MAMP/WAMP 或 Docker

### 1.4 部署环境
- **操作系统**: Linux (Ubuntu 22.04/CentOS 8) 或 Windows Server
- **网络环境**: 内部局域网（无互联网连接）
- **安全考虑**: 局域网防火墙，内部认证

## 3. Bootstrap本地化部署方案

### 3.1 资源下载与配置
由于系统在无互联网的局域网环境运行，所有外部资源必须本地化：

1. **Bootstrap 5.3.0** 下载到 `public/assets/css/bootstrap.min.css` 和 `public/assets/js/bootstrap.bundle.min.js`
2. **Bootstrap Icons 1.11.0** 下载到 `public/assets/css/bootstrap-icons.css` 和 `public/assets/fonts/` 目录
3. **jQuery 3.6.0**（可选）下载到 `public/assets/js/jquery.min.js`

### 3.2 本地资源引用示例
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>收租管理系统</title>
    
    <!-- 本地化Bootstrap CSS -->
    <link href="/assets/css/bootstrap.min.css" rel="stylesheet">
    
    <!-- 本地化Bootstrap Icons -->
    <link href="/assets/css/bootstrap-icons.css" rel="stylesheet">
    
    <!-- 自定义CSS -->
    <link href="/assets/css/custom.css" rel="stylesheet">
</head>
<body>
    <!-- 页面内容 -->
    
    <!-- 本地化jQuery (可选) -->
    <script src="/assets/js/jquery.min.js"></script>
    
    <!-- 本地化Bootstrap JS -->
    <script src="/assets/js/bootstrap.bundle.min.js"></script>
    
    <!-- 自定义JS -->
    <script src="/assets/js/app.js"></script>
</body>
</html>
```

## 4. 数据库设计（MariaDB）

### 4.1 数据库配置
```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS easy_rent DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE easy_rent;

-- 设置字符集
SET NAMES utf8mb4;
```

### 4.2 核心数据表设计
与之前的设计保持一致，但适配MariaDB语法：
```sql
-- 用户表
CREATE TABLE users (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE,
    phone VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    full_name VARCHAR(100),
    avatar VARCHAR(255),
    role ENUM('admin', 'landlord', 'tenant') DEFAULT 'tenant',
    is_active BOOLEAN DEFAULT TRUE,
    last_login_at TIMESTAMP NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_role (role),
    INDEX idx_email (email),
    INDEX idx_phone (phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 房产表
CREATE TABLE properties (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    owner_id INT UNSIGNED NOT NULL,
    title VARCHAR(200) NOT NULL,
    -- 其他字段...
    FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_owner (owner_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 5. 无网络环境适配方案

### 5.1 外部服务替代方案
| 外部服务 | 局域网替代方案 | 实现方式 |
|----------|----------------|----------|
| CDN资源 | 本地文件 | 下载Bootstrap、Icons到assets目录 |
| 在线支付 | 线下支付记录 | 手动标记支付状态，支持银行转账记录 |
| 短信通知 | 站内通知+邮件 | 局域网邮件服务器或纯站内通知 |
| 地图服务 | 静态地址 | 使用文本地址，无地图可视化 |
| 天气服务 | 无 | 不提供天气信息 |

### 5.2 功能调整
1. **支付功能**: 改为线下支付记录，支持上传支付凭证图片
2. **通知功能**: 仅支持站内通知，可选局域网邮件通知
3. **第三方登录**: 移除微信/支付宝登录，仅支持账号密码登录
4. **电子签名**: 改为上传扫描件或使用本地数字证书

### 5.3 数据导入导出
- 支持Excel/CSV格式数据导入
- 支持PDF/Excel报表导出
- 数据库备份与恢复功能

## 6. 安全设计（局域网环境）

### 6.1 网络安全
- 局域网防火墙配置
- IP白名单访问控制
- HTTPS自签名证书（可选）
- 定期安全扫描

### 6.2 应用安全
- SQL注入防护：使用预处理语句
- XSS防护：输出转义
- CSRF防护：Token验证
- 会话安全：Session固定防护
- 密码安全：bcrypt哈希

### 6.3 数据安全
- 数据库定期备份
- 敏感数据加密存储
- 访问日志记录
- 操作审计日志

## 7. 性能优化

### 7.1 前端优化
- Bootstrap和Icons本地化，减少HTTP请求

### 7.2 后端优化
- OPcache启用
- 数据库查询优化
- 页面缓存（Smarty或自定义缓存）
- 静态文件CDN替代为本地Nginx服务

### 7.3 数据库优化
- 合适的索引设计
- 查询优化
- 定期维护（优化表、清理日志）
