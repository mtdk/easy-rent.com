# 产品范围重评估（2026-04-06）

> 文档状态：历史（归档参考）
> 口径说明：以 docs/主任务看板.md 与 docs/任务清单.md 为准。


## 一、背景与约束

- 应用运行环境：单机/内网，不部署外网。
- 使用角色：管理员与房东为同一人（个人自用场景）。
- 核心目标：每月登记租客缴租情况、管理房屋租赁信息、维护租客资料与日常统计。

## 二、产品定位（收敛后）

- 本系统定位为：租赁台账与运营记录系统。
- 非目标：真实资金转账、在线支付清算、第三方支付网关集成。

## 三、必须功能（MVP）

1. 房屋管理：房屋信息、状态、备注。
2. 租客与合同管理：租客信息、合同生命周期、到期提醒。
3. 应收应付台账：月度应收生成、实收登记、逾期标记、滞纳金记录。
4. 统计报表：月度汇总、欠费清单、房屋出租情况。
5. 数据安全：本地备份与恢复、关键操作留痕（谁在何时修改）。

## 四、冻结/降级项（当前版本不做）

- 在线支付接口（微信/支付宝）对接。
- 支付渠道对账与结算链路。
- 外网通知能力（公网依赖）。
- 移动端独立应用。

## 五、保留扩展点（未来可扩）

- 预留支付渠道字段，保持数据结构兼容。
- 预留多角色权限扩展（tenant）。
- 预留导出接口（CSV/Excel）与对外 API。
- 预留多设备部署与外网接入开关。

## 六、执行策略（从复杂到简洁）

1. 文案统一：将“支付”语义调整为“收款登记/台账记录”。
2. 功能优先级：优先录入效率、准确性、统计可读性。
3. 测试优先级：覆盖登记边界、逾期计算、筛选统计，不扩展真实支付链路。
4. 发布优先级：先做稳定可用的单机版，再评估扩展模块。
